OK, fresh start. The issue is that the MP Builder prepends the Source hostname to all requests. Since Azure uses different hosts for auth vs data, we need a different approach.

Step 1 — Source:

Hostname: management.usgovcloudapi.net
Port: 443
SSL: Unverified
Base API Path: leave blank
Step 2 — Authentication:

Select Custom
Add fields:
Label	Sensitive?
tenant_id	No
client_id	No
client_secret	Yes
subscription_id	No
Step 3 — Session Authentication:

Toggle Yes
Get Session Request:

Method: POST
API Path: https://login.microsoftonline.us/${authentication.credentials.tenant_id}/oauth2/v2.0/token
Use the full URL here — this overrides the Source hostname for just this request.

Get Session Advanced:

Headers: Content-Type = application/x-www-form-urlencoded
Body:

grant_type=client_credentials&client_id=${authentication.credentials.client_id}&client_secret=${authentication.credentials.client_secret}&scope=https%3A%2F%2Fmanagement.usgovcloudapi.net%2F.default
Click Request — should return 200 with access_token.

Session Fields:

Check access_token
Release Session: Disabled

Global Request Headers:

Name: Authorization
Value: Bearer ${authentication.session.access_token}
Test Connection:

Method: GET
API Path: /subscriptions/${authentication.credentials.subscription_id}?api-version=2022-12-01
Click Request.
