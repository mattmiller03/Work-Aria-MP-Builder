# Aria Operations Management Pack — Azure Government Cloud

A custom VMware Aria Operations management pack that collects resource attributes from **Azure Government Cloud** using the VCF Operations Integration SDK (Python). Successfully tested against Aria Operations 8.18.6 Enterprise on an air-gapped network.

## What It Collects

| Resource Type | Count (production) | Key Attributes |
|---------------|-------------------|---------------|
| **Subscriptions** | 6 | ID, display name, state, tenant, quota |
| **Resource Groups** | 797 | Name, location, provisioning state, tags |
| **Virtual Machines** | 2,036 | VM size, power state, OS type, image, disks, NICs, security, zones |
| **Managed Disks** | 8,305 | SKU, size GB, IOPS, throughput, disk state, encryption |
| **Network Interfaces** | 4,650 | Private/public IPs, MAC, NSG, subnet, DNS, attached VM |
| **Virtual Networks** | 44 | Address prefixes, DNS, DDoS protection |
| **Subnets** | 335 | Address prefix, NSG, route table, service endpoints |
| **Storage Accounts** | 299 | Kind, SKU/tier, access tier, TLS, endpoints, encryption |
| **Load Balancers** | 31 | SKU, frontends, backends, rules, probes |
| **Key Vaults** | 86 | URI, SKU, soft delete, purge protection, RBAC |
| **SQL Servers** | 12 | FQDN, version, state, TLS, public network access |
| **SQL Databases** | 75 | SKU/tier/capacity, max size, collation, zone redundancy |
| **App Services** | 6 | Kind (web/function), state, host names, HTTPS-only |

**16,682 total objects** with **375,000+ properties** and full parent-child relationship hierarchy. All resource types also collect **tags** as dynamic properties.

### Dedicated Host Costing (v1.4.0)

Dedicated hosts include **hourly and monthly cost estimates** from the Azure Retail Prices API (with hardcoded fallback for air-gapped environments), VM size breakdowns, disk SKU tracking, and remaining capacity per VM size. Full parent-child relationships link Host Group > Dedicated Host > VM > Disk for drill-down dashboards.

See [dashboard build guide](docs/dashboard-dedicated-host-detail.md) for the per-host detail dashboard.

## Architecture

```
Azure/
├── Dockerfile                  ← Container build (FROM base-adapter:python-1.2.0)
├── commands.cfg                ← Maps HTTP endpoints to adapter subprocess commands
├── manifest.txt                ← Pack metadata (name, version, platform)
├── config.json                 ← Container registry configuration
├── connections.json.example    ← Credential template (connections.json is git-ignored)
├── eula.txt / icon.png
├── fetch_pricing.py            ← Standalone pricing updater (run from internet-connected PC)
├── app/
│   ├── adapter.py              ← Main entry point (collect, test, get_adapter_definition)
│   ├── constants.py            ← Adapter keys, Azure Gov endpoints, API versions
│   ├── auth.py                 ← OAuth2 client credentials flow (login.microsoftonline.us)
│   ├── azure_client.py         ← REST client with nextLink pagination + rate-limit retry
│   ├── helpers.py              ← SDK compatibility utilities (make_identifiers, safe_property)
│   ├── pricing.py              ← Azure Retail Prices API + fallback table for host costing
│   ├── collectors/             ← 18 per-resource-type collection modules
│   ├── requirements.txt
│   └── wheels/                 ← Bundled Python wheels for air-gapped Docker builds
├── conf/                       ← Generated by mp-build (describe.xml, describeSchema.xsd)
└── content/                    ← Dashboards and content (future)
```

**Collection flow:**

```
adapter.py collect()
  ├─ auth.py ── POST login.microsoftonline.us/{tenant}/oauth2/v2.0/token
  │              └─ Returns Bearer access_token (cached, auto-refreshed 5 min before expiry)
  ├─ azure_client.py ── GET management.usgovcloudapi.net/...
  │                      ├─ Follows nextLink pagination automatically
  │                      ├─ Retries on HTTP 429 (respects Retry-After header)
  │                      └─ Retries on 5xx with exponential backoff
  └─ collectors/* ── Each collector independently:
       ├─ Calls azure_client.get_all() for its resource type
       ├─ Creates Aria Operations objects with properties
       └─ Defines parent→child relationships
       (One collector failing doesn't block others)
```

## Azure Gov Service Principal Setup

1. Sign in to `https://portal.azure.us`
2. **Entra ID > App registrations > New registration** — name it `aria-operations-mp`
3. Note the **Application (client) ID** and **Directory (tenant) ID**
4. **Certificates & secrets > New client secret** — copy the **Value** (not the Secret ID)
5. **Subscription > IAM > Add role assignment** — assign **Reader** role to the app
6. Repeat for each subscription to monitor (or assign at management group level)

| Credential | What it maps to |
|-----------|----------------|
| App ID | `client_id` in connections.json |
| Tenant ID | `tenant_id` |
| Secret Value (password) | `client_secret` |
| Secret ID | Not used |

## Deployment Guide

See [deploy-to-new-server.md](deploy-to-new-server.md) for the full step-by-step deployment on a fresh MP Builder OVA. Key phases:

1. System packages and Python 3.12 build from source
2. Download offline packages (wheels, Docker images, PowerShell)
3. Install PowerShell and test Azure Gov connectivity
4. Install Aria SDK (with dependency version pinning)
5. Configure project and credentials
6. Test with `mp-test` (collect 16K+ objects)
7. Build `.pak` with `mp-build`
8. Set up local Docker registry with HTTPS certs
9. Upload `.pak` to Aria Operations
10. Configure adapter instance on Cloud Proxy

### Critical Air-Gapped Deployment Notes

These are hard-won lessons from deploying on an air-gapped Aria Ops 8.18.6:

| Issue | Solution |
|-------|---------|
| **`manifest.txt` platform** | Must be `["Linux Non-VA", "Linux VA"]` — bare `"Linux"` crashes the pak manager |
| **`.pak` is only 25KB** | Normal for container-based paks — the adapter image is pulled from a registry, not embedded |
| **`REGISTRY=` missing in `.conf`** | Aria Ops falls back to default `harbor-repo.vmware.com`. Set `use_default_registry: false` in `config.json` and specify full registry path |
| **Port 443 conflict** | Cloud Proxy's `config-modules-docker` container uses 443. Manually edit `.pak` to set `API_PORT=8080` and `API_PROTOCOL=http` |
| **Digest mismatch** | Rebuild `.pak` after pushing image to registry so digests align |
| **Registry HTTPS required** | Set up `registry:2` with self-signed TLS cert. Copy cert to Cloud Proxy at `/etc/docker/certs.d/<IP>:5000/ca.crt` |
| **Unsigned pak STIG** | No supported way to self-sign. Document exception with security team |

### Manually Fixing the .pak Port/Protocol

After `mp-build`, the adapter uses port 443/HTTPS by default which conflicts with existing Cloud Proxy containers:

```bash
# Extract the pak
cd /tmp && rm -rf azuregov-pak adapter-check
cp /path/to/AzureGovCloud_1.0.0.pak /tmp/azuregov.zip
unzip -o azuregov.zip -d azuregov-pak
unzip -o azuregov-pak/adapter.zip -d adapter-check

# Fix port and protocol
sed -i 's/API_PORT=443/API_PORT=8080/' adapter-check/AzureGovAdapter.conf
sed -i 's/API_PROTOCOL=https/API_PROTOCOL=http/' adapter-check/AzureGovAdapter.conf

# Repackage
cd adapter-check && zip -r /tmp/azuregov-pak/adapter.zip . && cd /tmp/azuregov-pak
zip -r /tmp/AzureGovCloud_1.0.0_fixed.pak .
```

## Customization

### Add/Remove Resource Types

Comment out collectors in `adapter.py`'s `collect()` function and remove the matching definition from `get_adapter_definition()`.

### Add Properties to a Collector

```python
# In the collector file:
safe_property(obj, "new_property", props.get("newField", ""))

# In adapter.py get_adapter_definition():
vm.define_string_property("new_property", "New Property")
```

### Change API Versions

Edit `Azure/app/constants.py`. Azure Gov lags 1-2 releases behind commercial Azure. Use 2023-xx versions:

```python
API_VERSIONS = {
    "virtual_machines": "2023-03-01",
    "disks": "2023-04-02",
    "network_interfaces": "2023-05-01",
    ...
}
```

### Switch to Commercial Azure

Set `cloud_environment` to `"commercial"` in `connections.json`. The adapter automatically switches all endpoints.

## Troubleshooting

| Issue | Solution |
|-------|---------|
| **401 on Azure login** | Wrong tenant/client/secret or expired secret |
| **502 Bad Gateway on Azure API** | API version too new for Azure Gov — lower it in `constants.py` |
| **Test connection timeout on Cloud Proxy** | Check Docker logs on Cloud Proxy; verify image is loaded and port isn't conflicting |
| **`.pak` upload "operation failed"** | Check `/storage/log/vcops/log/casa/casa.log` on Aria Ops for real error |
| **Cloud Proxy pulls from harbor-repo.vmware.com** | `.pak` conf missing `REGISTRY=` line — rebuild with explicit registry in `config.json` |
| **"mismatch on SuitePlatform enum"** | `manifest.txt` has `["Linux"]` — change to `["Linux Non-VA", "Linux VA"]` |
| **Container port 443 conflict** | Manually edit `.pak` conf to `API_PORT=8080`, `API_PROTOCOL=http` |
| **Collection returns empty** | Check adapter.log on Cloud Proxy; often a Python error in a collector |
| **"None for not nullable"** | Properties returning None — use `safe_property()` helper |
